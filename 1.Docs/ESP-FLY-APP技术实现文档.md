# ESP-FLY-APP 技术实现文档

## 文档说明

本文档描述 ESP-FLY Android APP 的架构设计、模块划分和技术实现方案。

---

## 1. 系统架构设计

### 1.1 整体架构

系统采用 **MVVM (Model-View-ViewModel) 架构模式**，结合 Jetpack Compose 实现声明式UI。通过 Kotlin Flow 实现 View 和 ViewModel 的自动同步。

```
┌─────────────────────────────────────────────────────────────┐
│                    View层 (Jetpack Compose)                   │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  主界面      │  │  设置界面    │  │  摇杆组件    │      │
│  │ MainScreen   │  │SettingsScreen│  │JoystickView  │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                            ↕ (数据绑定/Flow)
┌─────────────────────────────────────────────────────────────┐
│              ViewModel层 (视图模型层)                         │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ DroneViewModel (无人机控制ViewModel)                   │   │
│  │ - 飞行控制逻辑 / PID参数管理 / 连接状态管理           │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────────┐
│              Model层 (数据模型层)                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │ DroneUiState │  │ PidSettings  │  │  FlightData  │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────────┐
│              Network层 (网络通信层)                           │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ UdpClient (UDP通信客户端)                              │   │
│  │ - UDP Socket管理 / 数据包收发 / 协议解析分发           │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────────┐
│              Protocol层 (协议层)                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │ SimplePacket │  │ Commander    │  │PidConfigPkt  │      │
│  │ (协议编解码)  │  │   (0x01)     │  │   (0x02)     │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│  ┌──────────────┐  ┌──────────────┐                        │
│  │ BatteryPacket│  │PidResponsePkt│                        │
│  │   (0x82)     │  │   (0x83)     │                        │
│  └──────────────┘  └──────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 架构特点

1. **MVVM分离**：View、ViewModel、Model职责清晰，完全解耦
2. **声明式UI**：使用Jetpack Compose，UI随状态自动更新
3. **响应式编程**：通过Kotlin Flow实现数据流动
4. **协程支持**：使用Kotlin Coroutines处理异步操作

## 3. 模块职责说明

### 3.1 View层 (`ui/`)

**MainScreen** - 主控制界面
- 左侧：油门摇杆（垂直单向，0~100%）
- 右侧：姿态摇杆（Pitch/Roll双向）
- 中间：控制台监控区（PID参数，1Hz更新）
- 顶部：电池电压 + 连接状态 + 设置按钮

**SettingsScreen** - PID参数设置界面
- 3列卡片：Roll/Pitch/Yaw 角度环PID参数
- 发送按钮（一次性下发所有9个参数）

**JoystickView** - 摇杆组件
- THROTTLE模式：垂直单向（油门）
- FULL模式：双向（姿态控制）

### 3.2 ViewModel层 (`viewmodel/`)

**DroneViewModel** - 核心业务逻辑
- 管理UI状态（DroneUiState）
- 50Hz控制循环（摇杆 → 控制命令）
- 1Hz控制台打印（PID参数显示）
- PID参数管理（DataStore持久化）
- 连接状态管理
- 触摸释放自动归零

### 3.3 数据模型

| 模型 | 说明 |
| --- | --- |
| DroneUiState | UI总状态（连接、电压、飞行数据、控制台、PID设置） |
| PidSettings | 角度环PID设置（Roll/Pitch/Yaw × P/I/D = 9个参数） |
| PidParams | 单组PID参数（kp, ki, kd） |
| FlightData | 飞行数据（roll, pitch, yaw, thrust） |

### 3.4 Network层 (`network/`)

**UdpClient** - UDP通信客户端
- 广播模式发送控制命令
- 独立协程接收下行数据
- 通过Flow分发电池信息和PID响应
- 发送队列（LinkedBlockingQueue）异步发送

### 3.5 Protocol层 (`protocol/`)

协议实现与MCU端保持一致，详见：**[ESP-FLY通信协议接口文档](./ESP-FLY通信协议接口文档.md)**

| 类 | PacketID | 方向 | 说明 |
| --- | --- | --- | --- |
| SimplePacket | - | - | 协议编解码核心 |
| Commander | 0x01 | APP→MCU | 飞行控制命令（14B） |
| PidConfigPacket | 0x02 | APP→MCU | PID配置（36B，角度环9参数） |
| BatteryPacket | 0x82 | MCU→APP | 电池电压（4B） |
| PidResponsePacket | 0x83 | MCU→APP | PID参数响应（36B） |

## 4. 数据流设计

### 4.1 控制数据流（APP → MCU，50Hz）

```
用户操作摇杆 → ViewModel更新内部状态
    → 50Hz控制循环计算控制参数
    → Commander打包（0x01）
    → UdpClient广播发送 → ESP32
```

### 4.2 遥测数据流（MCU → APP，1Hz）

```
ESP32广播 → UdpClient接收 → SimplePacket解码
    ├─ 0x82 → BatteryPacket解析 → ViewModel → UI电池电压显示
    └─ 0x83 → PidResponsePacket解析 → ViewModel → 控制台PID参数打印
```

### 4.3 PID参数下发流程

```
用户修改PID参数 → 点击"发送"
    → PidConfigPacket打包（0x02，36B）
    → UdpClient发送 → ESP32应用新参数
    → ESP32回传PID响应（0x83） → 控制台确认
```
